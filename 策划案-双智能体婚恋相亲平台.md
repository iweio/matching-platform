# 双智能体婚恋相亲平台 — 竞赛策划案

---

## 一、项目概述

### 1.1 项目名称

**"双生·契合" — 双智能体自主婚恋撮合平台**

### 1.2 一句话定位

**智能体先谈恋爱，真人再奔现。** 每个人拥有一名专属 AI 数字化身，由双方智能体代替真人完成筛选、三观磨合与聊天测评，高度适配后才开放真人见面。

### 1.3 核心创新

传统婚恋平台面临三大困境：(1) 用户无效匹配多，时间成本高；(2) 初期沟通尬聊严重，真实性格难以展现；(3) 仅靠标签筛选，无法深度衡量三观契合度。

本平台的核心创新在于**将 AI Agent 从"辅助工具"升级为"数字代言人"**——用户的专属智能体基于人格蒸馏数据，能够以用户的说话风格、性格特征、恋爱模式和价值观体系，与对方智能体进行 30 轮自然对话。AI 先替你"相亲"，适配了再见真人。

## 三、核心功能模块

### 3.1 人格蒸馏系统

用户在注册后需要完成四步人格蒸馏问卷，采集 5 大维度数据：

| 维度 | 内容 | AI 用途 |
|------|------|---------|
| 说话风格 | 幽默风趣 / 温柔细腻 / 直接干脆 / 文艺清新 / 霸气外露 | 控制智能体语言风格 |
| 性格类型 | 外向开朗 / 内向文静 / 沉稳务实 / 活泼可爱 / 成熟稳重 | 影响对话节奏与回应方式 |
| 恋爱模式 | 浪漫型 / 务实型 / 陪伴型 / 独立型 / 粘人型 | 影响情感表达与期望 |
| 三观数据(JSON) | 婚姻观、消费观、家庭观、理想伴侣描述 | 价值观匹配计算的输入 |
| 雷点底线(JSON) | 最不能接受的行为与习惯 | 禁忌冲突检测的输入 |

蒸馏数据提交后，后端调用 Agent 服务的 `/model-refresh` 接口，将用户画像编译为专属 System Prompt，完成"数字分身"构建。

### 3.2 双智能体自主对话

**这是本平台最核心的功能。** 用户发起匹配后，后台异步启动双 AI 对话引擎：

- **对话轮数**：30 轮（可配置）
- **话题池**：8 大类别 40+ 子话题 — 兴趣爱好、工作生活、感情观念、家庭观念、未来规划、价值观、婚姻家庭、生活习惯
- **调度策略**：每会话独立追踪已讨论话题，避免全局串扰；按轮次分配话题，末期自动切换到总结模式
- **真实感保障**：
  - 每轮 prompt 注入双方的 `values_view` 与 `taboo`，引导对话围绕真实画像展开
  - 话题使用集合追踪，同会话不重复同类话题
  - 句式变换指令，避免机械化复读
  - LLM 不可用时自动降级为 200+ 行画像感知的模板回复

**前端体验**：通过 SSE（Server-Sent Events）实时推送每一轮对话，以聊天气泡形式逐条呈现。用户沉浸式观看双方智能体的完整交流过程。

### 3.3 五维适配评分引擎

对话完成后，Agent 服务调用 `/generate-report` 生成适配报告。**评分完全基于用户真实画像计算，不依赖随机数。**

**五个评分维度**：

| 维度 | 权重 | 计算依据 |
|------|------|---------|
| 情感契合度 | 25% | 消息互动平衡度(40%) + 性格相容矩阵(60%) |
| 价值观匹配度 | 25% | 三观关键词重叠检测（家庭观、消费观、婚姻观、理想伴侣） |
| 沟通舒适度 | 20% | 说话风格互补矩阵(60%) + 消息互动(40%) |
| 生活方式匹配度 | 15% | 恋爱模式兼容矩阵(50%) + 消息互动(50%) |
| 未来规划契合度 | 15% | 价值观共识度 + 多维度共同点加分 |

**三套核心相容矩阵**：

1. **性格相容矩阵**：`CHARACTER_COMPAT` — 12 条目，如 `("活泼","温柔"): 90`、`("幽默","沉稳"): 78`
2. **说话风格互补矩阵**：`SPEAK_STYLE_COMPAT` — 12 条目，如 `("幽默风趣","温柔细腻"): 95`、`("外向开朗","内向文静"): 92`
3. **恋爱模式兼容矩阵**：`LOVE_STYLE_COMPAT` — 6 条目，如 `("浪漫","务实"): 72`、`("务实","均衡"): 88`

**禁忌冲突检测**：扫描双方 `taboo` 字段中的 12 类禁忌（抽烟、喝酒、赌博、撒谎、懒惰、冷漠、自私、花心、暴力、抠门、妈宝、邋遢），对匹配的禁忌扣 12 分/条。

**个性化建议生成**：基于分值档位（≥85 / ≥70 / ≥55 / <55）给出差异化建议，并根据价值观共同点自动推荐互动话题，根据差异点给出沟通提醒。

### 3.4 实时风险检测

对话过程中，Agent 服务可对聊天记录进行 7 类风险模式扫描：

| 风险类别 | 检测方式 |
|---------|---------|
| 敷衍 | 短回复占比、响应延迟 |
| 冷漠 | 无追问、无情感词 |
| 情绪化 | 负面情绪词频 |
| 利己 | "我"字占比异常 |
| 套路化 | 模板句式匹配 |
| 不尊重 | 贬低、打断关键词 |
| 隐私侵犯 | 直接索要联系方式等 |

### 3.5 双向解锁机制

匹配报告生成后，双方各自查看报告并独立决定同意或拒绝。**只有双向同意才能解锁真人聊天。** 任一方拒绝，匹配立即终止。

前端通过 2 秒轮询检测对方决策状态，实时弹窗提醒：
- 对方拒绝 → Toast 提示"对方已拒绝匹配"，2 秒后自动返回
- 双方同意 → Toast 提示"双方已解锁"，1.5 秒后跳转聊天页

### 3.6 真人聊天与个人中心

- **聊天界面**：联系人列表 + 消息面板，支持 PC/移动端响应式布局
- **消息轮询**：3 秒间隔自动拉取新消息，临时消息乐观更新
- **个人中心**：用户信息卡片、匹配统计（次数/解锁数/成功率）、历史匹配记录
- **历史报告**：点击历史记录直接在弹窗中查看完整适配报告（雷达图 + 优势/风险/建议）

## 四、数据流与状态机

### 4.1 匹配状态流转

```
                    ┌──────────┐
                    │ 0: 匹配中  │
                    └────┬─────┘
                         │ 智能体对话启动
                    ┌────▼─────┐
                    │ 1: 对话中  │ ← SSE 实时推送对话
                    └────┬─────┘
                         │ 对话完成，报告生成
                    ┌────▼─────┐
                    │ 2: 报告已生成│
                    └────┬─────┘
                     ╱         ╲
              同意  ╱           ╲  拒绝
                 ╱               ╲
        ┌───▼───┐          ┌───▼───┐
        │5: 已解锁│          │4: 已终止│
        └───────┘          └───────┘
```

### 4.2 完整业务流程

```
用户注册 → 创建智能体 → 四步人格蒸馏 → 加载AI模型
  → 发起匹配 → 匹配池配对
    → 双智能体30轮自主对话 (SSE实时直播)
      → 五维适配评分 + 禁忌冲突检测
        → 生成适配报告 (总评分 + 优势分析 + 风险提示 + 婚恋建议)
          → 双向确认
            ├─ 双方同意 → 解锁真人聊天
            └─ 一方拒绝 → 匹配结束
```

## 五、创新点与竞争优势

### 5.1 创新点

1. **双智能体自主对话**：不是传统的"AI 推荐+真人聊天"，而是 AI 先替你完成筛选和磨合，节省大量无效社交时间
2. **人格蒸馏技术**：将抽象的"性格、三观、底线"编译为可执行的 Agent System Prompt，使 AI 真正以用户身份交流
3. **五维相容矩阵算法**：基于心理学的性格互补理论，自建 3 套相容矩阵 + 12 类禁忌检测，评分有据可依，非随机生成
4. **LangGraph 状态图编排**：四个 AI 管线（模型刷新、对话启动、报告生成、风险检测）以有向图形式编排，节点可独立测试、可复用
5. **全链路实时体验**：SSE 推送对话 → 雷达图可视化报告 → 实时双向决策检测，用户全程感知匹配进度
6. **LLM 降级容错**：LLM 不可用时自动切换到基于画像模板的确定性回复，保证系统可用性

### 5.2 与传统婚恋平台对比

| 维度 | 传统平台 | 本平台 |
|------|---------|--------|
| 筛选方式 | 标签匹配（年龄、身高、收入） | AI 人格蒸馏 + 三观深度对齐 |
| 初期沟通 | 真人尬聊，高社交压力 | AI 先聊 30 轮，零压力 |
| 匹配质量 | 量多质低，刷不完 | 质优量精，适配才解锁 |
| 隐私保护 | 真人资料直接暴露 | AI 代理交流，真人身分匿名 |
| 时间成本 | 海量右滑 + 无效对话 | 一次蒸馏，AI 代劳筛选 |
| 评分依据 | 用户自填标签 | 5 维相容矩阵实际计算 |

## 七、技术难点与解决方案

### 7.1 智能体对话的真实感

**难点**：LLM 生成的对话容易机械化、模板化，多轮后重复明显。

**方案**：
- 每会话独立话题追踪（避免全局串扰）
- 将用户画像（`values_view`、`taboo`）注入每轮 prompt，引导个性化表达
- 明确禁止使用固定开场白（如"很高兴认识你"）
- LLM 不可用时降级到 200+ 行画像感知的模板回复系统
- 消息种子引入时间熵，防止相同输入产生相同输出

### 7.2 匹配评分的可信度

**难点**：LLM 直接评分存在幻觉风险，随机数评分无意义。

**方案**：
- `_fallback_analysis()` 始终基于真实画像计算分数
- 三套相容矩阵覆盖性格、风格、恋爱模式
- 价值观比对使用关键词重叠检测，而非 LLM 主观判断
- 禁忌冲突逐条扫描 + 量化扣分
- LLM 仅用于润色文本展示（非决定评分），失败自动回退到计算文本

### 7.3 一方拒绝后的状态同步

**难点**：A 方拒绝后 B 方仍停留在报告页无法退出。

**方案**：
- 前端 2 秒轮询 `getProgress` 获取实时状态
- 检测到 `status=4`（已终止）后弹窗提示并自动跳转
- 后端 `MatchProgressResponse` 增加 `myOp` 字段，前端据此控制按钮显示
- SSE 流中也增加 `done` 事件的拒绝状态通知

- **群组匹配**：支持多人社交场景（如周末出游搭子）
- **智能体进化**：基于用户历史匹配反馈，在线微调 Agent 策略

### 11.3 长期愿景

- **去中心化身份**：用户拥有自己的 Agent NFT，可在不同婚恋平台间迁移
- **联邦学习匹配**：跨平台数据不出域，联合训练匹配模型
- **元宇宙约会**：智能体在虚拟空间中代表用户完成初步交流与互动
